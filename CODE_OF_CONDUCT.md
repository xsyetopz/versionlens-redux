# Code of Conduct

This project expects contributors to keep discussions useful, respectful, and safe for people with different backgrounds, skill levels, identities, and opinions.

This code merges the practical expectations used by the Rust project with the Microsoft Open Source Code of Conduct used by TypeScript projects. References:

- Rust Code of Conduct: https://www.rust-lang.org/policies/code-of-conduct
- Microsoft Open Source Code of Conduct: https://opensource.microsoft.com/codeofconduct/

## Expected behavior

Contributors are expected to:

- be kind, patient, and constructive;
- respect different experience levels and technical opinions;
- focus critique on code, design, behavior, and evidence;
- accept responsibility for mistakes and correct them plainly;
- help keep issue threads, pull requests, reviews, and chat spaces productive;
- respect privacy and avoid publishing private communication or personal information without consent.

## Unacceptable behavior

The following behavior is not acceptable in project spaces:

- harassment, intimidation, stalking, or sustained disruption;
- hateful, degrading, discriminatory, or exclusionary comments;
- sexualized language, imagery, or attention;
- threats of violence or encouragement of self-harm;
- doxxing or publication of private information;
- bad-faith review behavior, including repeated personal attacks or deliberate derailment;
- spam, trolling, or other conduct that maintainers determine is harmful to the project.

## Scope

This code applies in project repositories, issues, pull requests, review comments, discussions, release channels, and any public space where someone is representing the project.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, temporarily block participation, or permanently ban participants when needed to protect the project and its contributors.

Enforcement should be proportional, documented when practical, and focused on stopping harmful behavior rather than winning arguments.

## Reporting

Report conduct concerns to the repository maintainers through a private maintainer channel when one is available. If no private channel is available, open a GitHub issue with minimal public detail and ask for a private contact path.

Do not publish private personal information, screenshots of private messages, or sensitive allegations in public issue text unless the affected people have consented.
